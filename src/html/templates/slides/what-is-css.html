Cascading Style Sheets

CSS

  • Describes how an HTML document should display visually
  • Cascading describes how rules from one style sheet can override previously defined rules
  • Browsers use the order in which style sheets are included in the document and special selector specificity rules to determine which styles should be displayed

CSS

  • Style sheets are composed of rules
  • Each rule (or ruleset) is composed of one or more selectors and a block of property declarations
  • Selectors declare which part of the document a style applies to by matching tags, classes, and attributes in the document
  • < a href="https://en.wikipedia.org/wiki/Cascading_Style_Sheets#Declaration_block" target="_blank" title="Wikipedia.org - CSS Declaration Block" > Properties describe which aspects of an element should be styled and how

A few CSS rules


/* match elements by tag */
button {
  border: 0;                             /* remove border */
  border-radius: 4px;                    /* add rounded corners */
  box-shadow: 5px 5px rgba(0, 0, 0, .2); /* add faint shadow */
  cursor: pointer;                       /* make mouse cursor a finger */
  height: 100%;                          /* make height 100% of parent */
  width: 20%;                            /* make width 20% of parent */
}
/* match element by id */
#large-cta {
  font-size: 2em;                        /* make text 2x the normal size */
}
/* match elements by class */
.call-to-action {
  background-color: red;                 /* make the background red */
  color: white;                          /* make text white */
}
